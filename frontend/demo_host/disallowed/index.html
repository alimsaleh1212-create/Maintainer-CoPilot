CSP blocks the
widget iframe

This page demonstrates the security boundary. The widget embed script runs, but the browser enforces Content-Security-Policy: frame-ancestors and refuses to display the iframe.

How the block works
When the GET /embed?widget_id=... response is served, the API reads the widget's allowed_origins from the database and sets:

Content-Security-Policy: frame-ancestors http://localhost:8090

Because this page is served from port 8091 (not in the allowlist), the browser refuses to render the iframe and logs a CSP violation to the console.
